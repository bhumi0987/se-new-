Unit 4 — System Test & Quality

Testing strategies, QA practices, and metrics that ensure software meets requirements and quality goals.

Important Topics

  1. Testing Levels (unit, integration, system, acceptance)
  2. Test Design Techniques (equivalence, boundary, state-based)
  3. QA Metrics & KPIs (coverage, defect density)
  4. Automation & CI (test automation, pipelines)
  5. Defect Lifecycle & Reporting

GATE Question (reference)

Sample GATE-style question:

“Explain the differences between unit testing and system testing and when each is used.”

Reference / further reading: More reading on GeeksforGeeks

Very Short Questions

  • What is unit testing?
  • What does 'CI' stand for?
  • Define 'coverage' in testing.
  • Name one test automation tool.

Short Questions

  1. Explain why test automation matters.

    Automation speeds feedback, enables frequent regression checks, and reduces manual error for repetitive tests.

  2. Describe how acceptance testing is used with stakeholders.

    Acceptance tests verify the system meets business needs; stakeholders confirm acceptance criteria are met before release.

  3. How do you measure test effectiveness?

    Metrics like defect detection rate, escaped defects, and coverage help measure effectiveness.

  4. Give an example of a defect lifecycle step.

    A defect is reported → triaged → assigned → fixed → verified → closed.

MVP / High-yield Questions

  • Explain the difference between unit and system testing in one paragraph.
  • List three metrics to track test effectiveness and a limitation for each.
  • Describe a minimal automation strategy for continuous releases.
  • Give three characteristics of a good test case.
