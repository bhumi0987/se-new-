GATE Question (reference)

Sample GATE-style question:

“List and briefly describe common strategies for migrating legacy software to modern platforms.”

Reference / further reading: More reading on GeeksforGeeks

Very Short Questions

  • What is technical debt?

    • Re-hosting moves an application to new infrastructure with minimal code changes; rewriting replaces or reimplements the application core, which is costlier but may remove legacy constraints.

  • What does 'refactor' mean?
  • Give one reason to re-engineer software.

    • Incremental migration validates each small change and allows rollback if problems appear; it avoids a large single-point failure and spreads risk over time.

Testing preserves behavior, helps detect regressions introduced during refactors, and provides confidence when replacing components.

    Long Questions

    1. Design a migration plan for a legacy inventory system used by a small retailer, including tests and rollback strategies.
    2. Critically discuss the trade-offs between starting a rewrite vs refactoring an existing system.
    3. Explain how to build characterization tests for an undocumented legacy module.
    4. Propose a monitoring approach to validate migration success post-deployment.

    Migrating in small steps allows validation at each stage and avoids a single big-bang failure.

  2. What role does testing play during re-engineering?

    Testing ensures behavior is preserved and helps spot regressions introduced during changes.

  3. How can documentation help in reverse engineering?

    Good documentation speeds understanding of architecture and reduces guesswork when reconstructing design intent.
